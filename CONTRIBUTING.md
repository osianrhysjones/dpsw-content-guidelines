# Contributing to these guidelines

Thank you for helping keep this wiki accurate and useful.

## The easiest way — editing in GitHub

You do not need to install anything. You can edit pages directly in your browser.

1. Find the page you want to edit using the navigation on the left
2. Click the **Edit on GitHub** link (or navigate to the file in the repo)
3. Click the **pencil icon** (Edit this file) in the top right
4. Make your changes in the text editor
5. Scroll down to **Propose changes**, add a short note about what you changed, and click **Propose changes**
6. Click **Create pull request**

A member of the content team will review your suggestion and merge it if it looks good.

## If you use VS Code

1. Clone the repo to your computer
2. Create a new branch: `git checkout -b your-branch-name`
3. Edit the relevant `.md` file
4. Commit your changes and push the branch
5. Open a pull request on GitHub

## Markdown basics

Pages are written in Markdown. Here are the essentials:

```
# Heading 1
## Heading 2

**Bold text**

- Bullet point
- Another bullet

[Link text](https://example.com)
```

## What to do if you're not sure

Open a [GitHub Issue](../../issues) describing what you think needs changing,
and the content team will pick it up.
